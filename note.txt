택타일 센서 데이터 활용 전략

택타일 센서 자체에는 특별한 알고리즘이 내장되어 있지 않으며, 핵심은 센서 데이터를 어떻게 활용하느냐의 문제입니다. 택타일 센서는 다음 세 가지 목적으로 활용됩니다:

학습 데이터 수집 단계에서의 활용: 텔레오퍼레이션 중 각 손가락의 힘 균형 상태를 실시간으로 오퍼레이터에게 피드백하여 양질의 학습 데이터 수집을 지원합니다. 이상적으로는 햅틱 글로브 등의 리드 디바이스를 통해 로봇 손의 접촉감을 오퍼레이터가 직접 느낄 수 있어야 합니다.

모델 추론 시 상태 판단: 학습된 모델이 동작할 때 실시간 입력 데이터로 활용됩니다. 예를 들어, 3개 손가락 모두에서 접촉이 감지되어야 "성공적인 파지"로 판단하도록 학습된 경우, 이 조건을 실시간으로 확인하는 데 사용됩니다.

예외 상황 감지 및 안전 제어: 예상과 다른 센서 반응(예: 3개 손가락 중 2개에서만 접촉 감지)이 나타나면 모델의 잘못된 동작으로 판단하여 시나리오 중단이나 사용자 경고 등의 안전 장치를 작동시킵니다.

모방학습 데이터 구성 및 수집

하나의 에피소드는 다음 요소들이 프레임 단위로 동기화된 데이터셋으로 구성됩니다:

- 멀티 카메라 영상 (RGB, 필요시 Depth 포함)
- 로봇 팔 및 손가락의 관절 각도값
- 택타일 센서값 (접촉 여부, 압력 분포)
- 오퍼레이터의 제어 입력값

에피소드 수에 대한 절대적 기준은 없으며, 일반적으로 300-400개 정도가 기준이 되지만 100개만으로도 충분한 경우도 있고 수백 개를 수집해도 부족한 경우도 있습니다. 결정적인 요인은 작업의 난이도, 오퍼레이터의 숙련도, 그리고 무엇보다 데이터의 품질입니다.

커넥터 삽입 같은 정밀 작업의 경우: 카메라로는 포착하기 어려운 0.수 mm 단위의 정밀함이 필요하므로, 시각 정보가 가려진 상태에서도 손가락 끝의 택타일 센서를 통해 탐색하며 삽입하는 행동을 학습시킬 수 있습니다. 이때는 Depth 정보와 함께 각 손가락별 힘 센서 데이터가 필수적입니다.

AI 모델 선택 및 학습 전략

기존 검증된 파운데이션 모델을 활용하는 것이 현실적입니다. 간단한 실험에는 ACT(Action Chunking with Transformers)를, 본격적인 멀티모달 데이터를 다루는 경우에는 NVIDIA GROOT나 OpenVLA 같은 모델을 권장합니다.

자체 모델 개발은 ChatGPT를 처음부터 만드는 것과 유사한 규모의 작업이므로 권장되지 않습니다. 대신 기존 모델의 입력 벡터 구조를 확장하여 Depth나 택타일 데이터를 추가하는 방식의 커스터마이징이 효과적입니다.

주의사항: 새로운 모달리티(센서 데이터)를 추가할 경우, 기존 pretrained weight 위의 파인튜닝이 아닌 아키텍처 레벨의 변경이 필요하므로 백본부터 재학습해야 합니다. 실제 경험 기준으로 H100 8장으로 약 일주일 정도의 학습 시간이 소요됩니다.

시뮬레이션의 한계: Isaac Sim 등을 활용하고 싶지만, 현실적으로 시뮬레이션에서 학습된 모델은 시뮬레이션에서만 잘 동작하는 경향이 있어 실환경 적용이 어렵습니다. 따라서 실제 로봇에서의 데이터 수집이 우선되어야 합니다.

하드웨어 아키텍처 및 통신 설계

고수준 제어 및 AI 추론은 메인 PC에서 통합적으로 처리하는 것이 적절합니다. 핸드나 개별 로봇 컴포넌트가 독립적인 추론 기능을 가질 필요는 없으며, 최소한의 로컬 안전 기능(과전류, 비상정지 등)만 하위 레벨에서 처리하면 됩니다.

통신 프로토콜: CAN 통신이 기본적으로 견고하고 안정적인 선택입니다. 필요에 따라 EtherCAT 등 고속 통신도 고려할 수 있으나, 범용성과 사용자 환경을 고려할 때 CAN이 더 실용적입니다. 핵심은 모든 하드웨어 통신이 하나의 OS/메인 시스템에서 통합 관리되는 구조를 만드는 것입니다.

팜 카메라 및 추가 센서: 손바닥이나 손목에 장착되는 카메라의 경우, 데이터량이 크기 때문에 핸드 내부에서 모든 처리를 하기보다는 raw 데이터를 메인 PC로 전송하여 처리하는 것이 일반적입니다. 다만 기본적인 제스처(주먹 쥐기, 컵 잡기 등)는 핸드에 사전 정의해 둘 수 있습니다.

데이터 수집 방법론: 텔레오퍼레이션 vs 모션캡처

텔레오퍼레이션이 모션캡처보다 우선되어야 합니다. 모션캡처는 오퍼레이터가 자연스럽게 동작할 수 있다는 장점이 있지만, 다음과 같은 근본적인 문제들이 있습니다:

물리적 제약 불일치: 사람의 동작이 로봇이 물리적으로 수행할 수 없는 속도나 자세일 수 있으며, 이를 실시간으로 판별하기 어렵습니다.

카메라 시야 문제: 사람이 박스를 드는 자세가 로봇의 카메라 시야를 가리거나 토크 한계를 초과하는 등 실제로는 사용할 수 없는 데이터가 될 수 있습니다.

검증 비용: 결국 모션캡처 데이터도 실제 로봇으로 리플레이하여 검증해야 하므로 이중 비용이 발생합니다.

따라서 처음부터 실제 로봇을 텔레오퍼레이션하여 "로봇이 실제로 따라할 수 있는" 데이터를 수집하는 것이 가장 신뢰성 높고 효율적인 방법입니다.

핵심 권장사항 요약

1. 택타일 센서는 학습-추론-안전의 3단계 모두에서 핵심적으로 활용
2. 햅틱 피드백이 가능한 고성능 리드 디바이스 구축이 데이터 품질의 핵심
3. 기존 파운데이션 모델(GROOT, OpenVLA 등) 활용 및 커스터마이징 전략 채택
4. 메인 PC 기반 통합 제어 아키텍처로 설계
5. 텔레오퍼레이션 기반 데이터 수집 방법 우선 채택
6. CAN 등 검증된 통신 프로토콜을 활용한 단일 OS 환경 구성

이러한 전략을 바탕으로 로봇 핸드 시스템을 설계하면 효과적인 모방학습과 안정적인 텔레오퍼레이션이 가능한 시스템을 구축할 수 있을 것입니다.
